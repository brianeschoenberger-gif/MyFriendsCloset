# PRODUCT_SPEC.md

## App name

My Friends Closet

## One-line description

A private social wardrobe app where friends can share, borrow, and style each other's clothes.

## Positioning

Shop your friends' closets before you buy something new.

## Target user

Initial target:
- Women 18-35 with active social calendars
- Friend groups planning weddings, trips, events, parties, or nights out
- Roommates, sisters, college friends, and close friend groups who already borrow clothes informally

Secondary users include couples, families, fashion-conscious men, event groups, and college students.

## Core user jobs

1. Upload my clothes so I can see my closet visually.
2. Choose which clothes my friends can see.
3. Mark some clothes as borrowable.
4. Browse my friends' borrowable clothes.
5. Request to borrow something.
6. Get AI outfit suggestions for an occasion.
7. Let friends help style me.
8. Plan outfits for a trip or event.

## MVP feature list

### Must-have

- Landing page and app dashboard
- Closet grid and add item flow
- Item visibility controls and borrowable toggle
- Friend closet browsing and borrow request flow
- Style Me outfit generator using mock AI
- Event Closet page, mock data, and responsive design

### Should-have

- Filters by category, color, size, friend, occasion, and borrowable status
- Item detail view, outfit cards, friend avatars, request statuses, empty states, demo onboarding, and friend voting

### Later

- Auth, database, image upload, AI item tagging/background removal/outfit generation, friend invites, notifications, calendar/weather integration, and any financial or damage flows

## Required screens

- Landing: product explanation, how it works, sharing, borrowing, styling, and CTA.
- Dashboard: closet counts, requests, friend activity, Style Me CTA, and upcoming event.
- My Closet: item grid, add action, filters, and visibility indicators.
- Add Item: image placeholder, item details, tags, visibility, borrowable state, and notes.
- Friends' Closets: friend list, shared grid, filters, and borrow actions.
- Borrow Requests: incoming/outgoing requests, statuses, and approve/decline actions.
- Style Me: occasion, vibe, optional location, closet source, and three explained outfits.
- Event Closet: event details, members, shared items, and outfit suggestions.

## UX tone

Friendly, stylish, casual, and smart. Prefer copy such as "Want to borrow this?", "Style me for Saturday", "Your closet, but social", "Only share what you choose", and "Borrow before you buy."
